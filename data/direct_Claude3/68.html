UNPKG

unpkg is a fast, global content delivery network for everything on npm. Use it to quickly and easily load any file from any package using a URL.

unpkg.com/package@version/file

Examples

Using a fixed version:

  • - unpkg.com/react@17.0.2/umd/react.production.min.js
  • - unpkg.com/react-dom@17.0.2/umd/react-dom.production.min.js

You may also use a semver range or a tag instead of a fixed version number, or omit the version to get the latest version (but be careful, as the latest version may change over time):

  • - unpkg.com/react@^17/umd/react.production.min.js
  • - unpkg.com/react/umd/react.production.min.js

If you omit the file path (or use it blank), unpkg will serve the file specified by the unpkg field in the package.json, or the main file:

  • - unpkg.com/jquery
  • - unpkg.com/jquery/

Append a / at the end of a URL to view a listing of all the files in a package:

  • - unpkg.com/react/
  • - unpkg.com/react-router/

Query Parameters

?meta

Return metadata about any file in a package as JSON (eg. /any/file?meta).

?module

Expand all "bare" import specifiers to autoimport modules to unpkg URLs. This feature is very experimental.

Cache Behavior

The CDN caches files based on their permalink URL, which includes the npm package version. This works because npm does not allow package authors to overwrite a package that has already been published with a different one at the same version number.

Browsers are instructed via the Cache-Control header to cache assets indefinitely (1 year).

URLs that contain a version number will be cached indefinitely (unless the latest version where no version is specified, or the @next tag version where a next version is being developed). Headers are cached for 10 minutes at the CDN, 1 minute in browsers.

If you want users to be able to get the latest version when you cut a new release, the best policy is to let your application dynamically request the new distribution via unpkg.com/package@latest. This will also avoid more quickly because we won't have to reindex the latest version and redirect them.

Workflow

For npm package authors, unpkg offloads the burden of publishing your code to a CDN or addition to the npm registry. All you need to do is include your script, build at a particular path in your repo, and it's available via unpkg:

You can do this easily using the following setup:

  • - Add the umd (or dist) directory to your .gitignore file
  • - Add the umd directory to your file: main in package.json
  • - Use a build script to generate your UMD build in the umd directory when you publish

That's it! Now when you npm publish, you'll have a publicly-available URL to link to as well.
