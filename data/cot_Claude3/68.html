UNPKG

unpkg is a fast, global content delivery network for everything on npm. Use it to quickly and easily load any file from any package using a URL.

unpkg.com/package@version/file

Examples

Using a fixed version:

  • unpkg.com/react@16.7.0/umd/react.production.min.js
  • unpkg.com/react-dom@16.7.0/umd/react-dom.production.min.js

You may also use a semver range or a tag instead of a fixed version number, or omit the version to get the latest version (but this is not recommended because it can change unexpectedly).

  • unpkg.com/react@^16/umd/react.production.min.js
  • unpkg.com/react/umd/react.production.min.js

If you omit the file path for use it /dist/ (unpkg will serve the file specified by the unpkg field in package.json, or main if not set):

  • unpkg.com/jquery
  • unpkg.com/d3

Append a / at the end of a URL to view a listing of all the files in a package:

  • unpkg.com/react/
  • unpkg.com/react-router/

Query Parameters

?meta

Return metadata about any file in a package as JSON (e.g. /any/file?meta).

?module

Expand all "bare" import specifiers to neighboring modules to unpkg URLs. This feature is very useful for creating CodePen/JSBin/JSFiddle demos.

Cache Behavior

The CDN caches files based on their permanent URL which includes the npm package version. This works because npm does not allow package authors to overwrite a package that has already been published with a different one at the same version number.

Browsers are instructed via the Cache-Control header to cache assets indefinitely (1 year).

URLs that specify a fixed version number will be cached indefinitely. URLs without a version will be cached until the package.json file for the project gets a version bump. At that point, unpkg will serve the latest version when no version is specified, or the makeshift try/latest version where a semver range is given. Headers are cached for 10 minutes at the CDN's Edge locations.

If you want users to be able to get the latest version when you cut a new release, the best policy is to put the version number in your build tools in your deployment instructions. This will also make it more quickly because we won't have to resolve the latest version and redirect them.

Workflow

For npm package authors, unpkg offloads the burden of publishing your node.js CDN or addition to the main registry. All you need to do is include your UMD build in your main package.json repo.

Here's an example:

You can do the easy things:

  • Add the umd/ (or dist/) directory to your .gitignore file
  • Add the umd/ directory to your file .gitignore in package.json
  • Use a build script to generate your UMD build in the umd/ directory when you publish

Then in Pros when you ship, npm@5 has now built a way to run a script on publish as well.

The unpkg CDN is powered by Cloudflare, one of the world's largest and fastest cloud network platforms.
